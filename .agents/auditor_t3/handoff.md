## 2. Logic Chain

- The test suite queries an in-memory SQLite database initialized with the exact production Drizzle schema (Drizzle SQL migration script execution during `setupTestDb`).
- When tests issue requests to the mock endpoint `/organizations`, `/bucket/tenant_*`, `/api-keys/revoke`, `/database/seed`, and `/workflows/...`, the `fetchWrapper` runs real database read/write queries and R2/Sentry mock logic.
- Assertions dynamically verify:
  1. The API key generated by `OrgOnboardingWorkflow` is stored in the database and works to authenticate subsequent requests.
  2. R2 uploads are restricted to a maximum of 2 files under the `"prod-basic"` plan, and Sentry triggers warnings on violations.
  3. Revoking an API key updates the database record state to `'revoked'` and denies access, while session tokens remain valid.
  4. Database seeding fills tables correctly, and RBAC rules block members from inviting users or deleting organizations.
  5. Workflow exceptions and details are pushed onto Sentry arrays with correct tags (e.g. `workflowName`, `stepName`), and retries recover the status state.
- Because all checks run on dynamic outputs and require valid SQLite operations to pass, they represent authentic testing logic rather than hardcoded shortcuts or facade code.
