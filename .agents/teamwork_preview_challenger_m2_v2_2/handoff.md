## 2. Logic Chain

- **Cookie Bypass Check**:
  - In `api-key.ts`, the short-circuit condition checks like `if (c.get("user")) { await next(); return; }` are absent.
  - Regardless of whether the global session middleware (lines 65-86 of `apps/data-service/src/index.ts`) resolves a cookie session and sets `c.get("user")`, `requireApiKey` continues to extract the API key from headers.
  - If headers do not contain a key, it returns 401 with `"API key is missing"`. If the key is invalid, it returns 401 with `"Invalid API key"`.
  - Thus, cookie sessions cannot bypass API key validation on protected endpoints.
- **Organization Context Mapping**:
  - The middleware extracts `result.key.referenceId` (representing the organization ID linked to the developer API key).
  - It sets `session` to `{ activeOrganizationId: result.key.referenceId }`.
  - Resource routes (e.g. `listDomainsHandler` in `apps/data-service/src/endpoints/domains/list.ts`) extract `activeOrganizationId` from `c.get("session")` and use it to filter database queries (e.g. `listDomains(db, organizationId)`).
  - Therefore, database queries are correctly and securely scoped to the organization associated with the API key.
- **Edge Case Robustness**:
  - Empty API keys are trimmed and evaluated as falsy, triggering a 401 `"API key is missing"` response.
  - Strings like `"null"` or `"undefined"` are not valid keys, so they fail verification in `verifyApiKey`, returning 401 `"Invalid API key"`.
  - Parametrized SQL statements generated by Drizzle ORM in Better Auth's key lookup prevent SQL injection vectors completely.

## 3. Caveats

- **Mocked Better Auth API**: In tests, Better Auth API's `getSession` and `verifyApiKey` functions are mocked because the physical Better Auth server is not running during local unit test suites. However, the mocked behaviors accurately simulate the exact request/response format of Better Auth.

## 4. Conclusion

- The implementation of Milestone 2 (R1) v2 API key authentication is robust, correct, and secure.
- There is no cookie bypass vulnerability.
- Scoping of requests to organization ID is implemented correctly, ensuring multi-tenant isolation.
- Vulnerabilities from edge cases (whitespace keys, null values, SQL injection) are successfully mitigated.
